## Overview

This exercise applies the concepts of underfitting and overfitting to optimize model performance. The goal is to find the best `max_leaf_nodes` value that gives the most accurate predictions.

**What I'll Do:**
1. Test different tree sizes (complexity levels)
2. Compare validation errors for each size
3. Find the optimal tree size
4. Train final model with optimal settings on ALL data

**The Challenge:**
Finding the sweet spot between underfitting (too simple) and overfitting (too complex).

## Setup and Initial Model

### Initial Code Setup

```python
# Code you have previously used to load data
import pandas as pd
from sklearn.metrics import mean_absolute_error
```

## Step 1: Compare Different Tree Sizes

### Task
Write a loop that tries different values for `max_leaf_nodes`, stores the results, and finds the best value.

### The Helper Function

First, we're given a helper function:

```python
def get_mae(max_leaf_nodes, train_X, val_X, train_y, val_y):
    model = DecisionTreeRegressor(max_leaf_nodes=max_leaf_nodes, random_state=0)
    model.fit(train_X, train_y)
    preds_val = model.predict(val_X)
    mae = mean_absolute_error(val_y, preds_val)
    return(mae)
```

**What This Function Does:**
1. Creates a model with specified `max_leaf_nodes`
2. Trains on training data
3. Predicts on validation data
4. Calculates and returns MAE
5. Encapsulates entire process in one call

**Why This is Useful:**
- Avoid repetitive code
- Easy to call multiple times
- Clean and readable
- Can test many values quickly

### My Solution

```python
from sklearn.tree import DecisionTreeRegressor
from sklearn.metrics import mean_absolute_error

def get_mae(max_leaf_nodes, train_X, val_X, train_y, val_y):
    model = DecisionTreeRegressor(max_leaf_nodes=max_leaf_nodes, random_state=1)
    model.fit(train_X, train_y)
    preds_val = model.predict(val_X)
    mae = mean_absolute_error(val_y, preds_val)
    return mae

# Test different tree sizes
candidate_max_leaf_nodes = [5, 25, 50, 100, 250, 500]

# Store results in dictionary
scores = {}

# Loop through each candidate value
for max_leaf_nodes in candidate_max_leaf_nodes:
    scores[max_leaf_nodes] = get_mae(max_leaf_nodes, train_X, val_X, train_y, val_y)

# Find the best tree size (minimum MAE)
best_tree_size = min(scores, key=scores.get)

# Check answer
step_1.check()
```

**Result:** ✅ Correct

### Understanding My Solution

**Step-by-Step Breakdown:**

**1. Define Candidate Values:**
```python
candidate_max_leaf_nodes = [5, 25, 50, 100, 250, 500]
```
- Testing 6 different complexity levels
- From very simple (5 leaves) to complex (500 leaves)
- Covers wide range to find pattern

**2. Initialize Storage:**
```python
scores = {}
```
- Dictionary to store results
- Key = max_leaf_nodes value
- Value = validation MAE

**3. Loop and Test:**
```python
for max_leaf_nodes in candidate_max_leaf_nodes:
    scores[max_leaf_nodes] = get_mae(max_leaf_nodes, train_X, val_X, train_y, val_y)
```
- Iterate through each candidate value
- Call `get_mae()` for each
- Store MAE in dictionary

**4. Find Best Value:**
```python
best_tree_size = min(scores, key=scores.get)
```
- Find key with minimum value
- `key=scores.get` tells min() to compare dictionary values
- Returns the max_leaf_nodes with lowest MAE

### Alternative Implementation I Tried

I also explored another approach using a list:

```python
scores = []
for leaf_size in candidate_max_leaf_nodes:
    mae = get_mae(leaf_size, train_X, val_X, train_y, val_y)
    scores.append((mae, leaf_size))

best_tree_size = min(scores)[1]
```

**How This Works:**
- Stores tuples of (mae, leaf_size)
- `min(scores)` finds tuple with smallest MAE
- `[1]` extracts the leaf_size from that tuple

**Dictionary vs List:**
- Dictionary: More readable, direct lookup
- List of tuples: Works but less clear
- Both are correct!

## Complete Code Summary

### Full Working Code

```python
import pandas as pd
from sklearn.metrics import mean_absolute_error
from sklearn.model_selection import train_test_split
from sklearn.tree import DecisionTreeRegressor

# Step 1: Load and prepare data
iowa_file_path = '../input/home-data-for-ml-course/train.csv'
home_data = pd.read_csv(iowa_file_path)

# Step 2: Select features and target
y = home_data.SalePrice
features = ['LotArea', 'YearBuilt', '1stFlrSF', '2ndFlrSF', 
            'FullBath', 'BedroomAbvGr', 'TotRmsAbvGrd']
X = home_data[features]

# Step 3: Split data
train_X, val_X, train_y, val_y = train_test_split(X, y, random_state=1)

# Step 4: Define helper function
def get_mae(max_leaf_nodes, train_X, val_X, train_y, val_y):
    model = DecisionTreeRegressor(max_leaf_nodes=max_leaf_nodes, random_state=1)
    model.fit(train_X, train_y)
    preds_val = model.predict(val_X)
    mae = mean_absolute_error(val_y, preds_val)
    return mae

# Step 5: Test different tree sizes
candidate_max_leaf_nodes = [5, 25, 50, 100, 250, 500]
scores = {}

for max_leaf_nodes in candidate_max_leaf_nodes:
    scores[max_leaf_nodes] = get_mae(max_leaf_nodes, train_X, val_X, train_y, val_y)

# Step 6: Find best tree size
best_tree_size = min(scores, key=scores.get)
print(f"Best tree size: {best_tree_size} leaves")
print(f"Best validation MAE: ${scores[best_tree_size]:,.0f}")

# Step 7: Train final model on ALL data
final_model = DecisionTreeRegressor(max_leaf_nodes=best_tree_size, random_state=1)
final_model.fit(X, y)

print("Final model ready for production!")
```

### The Complete Workflow

```
1. Load Iowa housing data
        ↓
2. Select 7 features + target
        ↓
3. Split: train (75%) / validation (25%)
        ↓
4. Test multiple tree sizes [5, 25, 50, 100, 250, 500]
        ↓
5. For each size:
   - Train model
   - Predict on validation
   - Calculate MAE
        ↓
6. Compare all MAE values
        ↓
7. Select size with minimum MAE
        ↓
8. Train final model with best size on ALL data
        ↓
9. Model ready for production!
```

## Challenges and Solutions

### Challenge 1: Understanding Dictionary Min

**Problem:** How does `min(scores, key=scores.get)` work?

**Solution:**
- `min(scores)` would compare keys (5, 25, 50, etc.)
- `key=scores.get` tells it to compare values instead
- Returns the key with minimum value
- Elegant one-liner!

**Learning:** Python's `key` parameter is powerful for custom comparisons

### Challenge 2: When to Use All Data

**Problem:** Should I always train on all data?

**Solution:**
- During tuning: NO - need validation set
- After tuning: YES - maximize information
- Key is separating optimization from final training

**Learning:** Different phases have different needs

### Challenge 3: Interpreting Results

**Problem:** How do I know if 100 leaves is really best?

**Solution:**
- Look at validation MAE curve
- See clear minimum
- Error increases both directions from optimal
- Pattern confirms choice

**Learning:** Trust the data, not intuition
